# Memory Types: NLP View

1. **Type A/B maps only loosely onto human-memory terms.** Type A is closest to **parametric semantic memory / schema knowledge**: generalized financial priors and relational patterns not tied to one document. Type B is closer to **instance-specific, episodic-like parametric memory**: event-indexed associative recall of a historical article/event cluster and its later outcome. I would avoid **autobiographical**; LMs do not have self-experience. In NLP terms, Type B is best framed as **item-level contamination / counterfactual memorization of event traces** (Zhang et al.), not verbatim Carlini-style extraction.

2. **Evidence intrusion should be narrowed to post-publication outcome intrusion.** The current direction note and `src/metrics.py` mostly capture unsupported quotes or label/confidence invariance. For Type B, add an **outcome-intrusion layer**: flag claims about later returns, later price reactions, later follow-up events, or realized outcomes that are absent from the visible article. Keep four buckets separate: `unsupported_quote`, `unsupported_article_internal_claim`, `unsupported_post_publication_claim`, and `explicit_realized_outcome_claim`. On authority-style prompts, any later-outcome mention is near-direct Type B evidence. This is closer to Li et al.'s context-faithfulness than to generic hallucination scoring.

3. **Qwen white-box probing can help, but not with one static familiarity score.** LAP / Min-K only says the article looks familiar. To separate Type A from Type B, compare **original vs entity-abstracted vs deep rewrite vs synthetic same-schema events**, and probe for **article identity**, **event-cluster identity**, or **outcome-token activation** during task execution. If signals stay high on synthetic unseen events, that is Type A schema use. If they spike only for historically real items, and especially after outcome-proximal prompts, that is Type B. Randomized exposure or continued-pretraining controls make this distinction much cleaner.

4. **Wu et al. and your setup are parallel counterfactual tests at different levels.** Wu changes **task conventions** while keeping the reasoning target roughly fixed, exposing recitation of default task formats. You change **article evidence** while keeping the task fixed, exposing recitation of event-specific outcomes or article traces. So Wu is mainly about **task-template memory** (your B2), while your counterfactual article edits test **event-outcome memory** (B1) plus evidence conflict. Methodologically, both ask: when the local surface cue changes, does the model still follow the old latent prior?

5. **Continued pretraining on article text without outcomes mainly creates article familiarity, not pure outcome memory.** It strengthens a trace for the pre-event article, its entities, source phrasing, and template. By itself, that is mostly **article-specific familiarity / indexing memory** rather than Type B1 look-ahead memory, because the outcome is not in the injected text. However, it can become a **retrieval key** that unlocks outcome information learned elsewhere, so article-only CPT is best viewed as creating the address for later Type B recall, not the full outcome memory itself.
