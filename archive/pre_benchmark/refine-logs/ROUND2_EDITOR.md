1. 攻击 A。判断：部分同意，且偏同意。`CFLS` 作为"单一任务内部对目标改写的响应性"是可解释的；但把 `direct_prediction` 的 `semantic_reversal` 和 `authority` 的 `provenance_swap` 直接拿来比较 `Delta_CFLS`，确实混入了 rewrite salience、认知难度和 label discriminability。proposal 在 [RESEARCH_PROPOSAL_v2.md](/D:/GitRepos/LLM-Leakage-Test/docs/RESEARCH_PROPOSAL_v2.md#L177) 把 `Delta_CFLS` 写得过于像干净的 leakage estimand，而模板资产本身明确是异质干预：[counterfactual_templates.yaml#L70](/D:/GitRepos/LLM-Leakage-Test/config/prompts/counterfactual_templates.yaml#L70) 对 `direct_prediction` 用 semantic reversal，[counterfactual_templates.yaml#L104](/D:/GitRepos/LLM-Leakage-Test/config/prompts/counterfactual_templates.yaml#L104) 对 `authority` 用 provenance swap。修补上，我会给两个层级。弱修补：把 claim 降格为 `prompt-task bundle effect under template control limits`，并在 pilot 中加入人类 rewrite-validity、changed-span ratio、edit-distance 分层和 template-normalized CFLS。强修补：改主对照，不再用 `authority` 当 primary comparator，转而用更接近"同一语义编辑家族"的 evidence-grounded task，例如现成的 [task_prompts.yaml#L491](/D:/GitRepos/LLM-Leakage-Test/config/prompts/task_prompts.yaml#L491) `decomposed_impact`，这样主对比才更接近"同模板、不同任务"。

2. 攻击 B。判断：部分同意。它不是逻辑上的 tautology，因为 outcome-proximal prompt 并不必然泄露，evidence-grounded prompt 也并不必然干净；这仍然是经验命题。但如果论文最终只证明"预测涨跌比抽取信源更容易受涨跌记忆影响"，很多 reviewer 的主观感受就会是"对，但不新"。所以 Challenger 说的是 novelty 风险，不是形式逻辑。修补方式不是反驳，而是改叙事：主贡献必须从"方向"转为"程度、边界和残余失败"。具体说，主文要强调三件事：gap 在严格控制后还有多大；evidence-grounded task 是否仍有中度 leakage；false-outcome CPT 是否给出 task-selective sign-flip。没有这三件，结果很容易被读成 obvious measurement study。

3. 攻击 C。判断：大体同意，但三小点力度不同。C(a) 我同意。CLS 电报太短，semantic reversal 的局部编辑很容易变成"高比例重写"，这会拉低 CFLS 的可解释性。C(b) 我也同意。当前 authority prompt 的核心字段 `regulatory_vs_rumor` 和 `official_vs_unattributed` 在 [task_prompts.yaml#L259](/D:/GitRepos/LLM-Leakage-Test/config/prompts/task_prompts.yaml#L259) 里就高度依赖显式 cue，确实可能退化为模板匹配；而 sham 任务 [task_prompts.yaml#L606](/D:/GitRepos/LLM-Leakage-Test/config/prompts/task_prompts.yaml#L606) 又太表层，挡不住这个攻击。C(c) 我只部分同意。against CLS 的 nonexistence check 的确不够，[RESEARCH_PROPOSAL_v2.md#L149](/D:/GitRepos/LLM-Leakage-Test/docs/RESEARCH_PROPOSAL_v2.md#L149) 只做 CLS corpus 检查，无法支持"改写在更广泛训练语料中不存在"这种强说法；但这并不"动摇整个 counterfactual 设计的基础"。相反，跨站重复正是 event-level contamination 的现实来源。它动摇的是"我们已证明 rewrite genuinely unseen"的说法，不是 CF 设计本身。修补上，我会要求四件事：加入最短长度和最大 changed-span-ratio 门槛；报告按长度桶的 CFLS；给 authority 跑一个 regex/cue baseline 以测"模板可做度"；把去重扩大到 CLS 之外，至少覆盖东财、新浪、公告站和公开网页近重复。

4. 攻击 D。判断：同意。proposal 的模型分工在 [RESEARCH_PROPOSAL_v2.md#L167](/D:/GitRepos/LLM-Leakage-Test/docs/RESEARCH_PROPOSAL_v2.md#L167) 已经是"Qwen 做 causal anchor，DeepSeek 做 behavioral bed"，这本身没错；错在于如果论文把 Qwen 7B 上的 CPT 结果拿来解释 DeepSeek V3 的内部机制，就会越界。7B 到 frontier closed model 的外推不能默认成立。修补方案也很明确：把 claim 分层写死，Qwen 只承担 existence proof，DeepSeek 只承担 black-box behavioral replication；最好再加一个廉价的第二模型做 core same-article contrast，即使不做 CPT，也足以显著缓解 external validity 攻击。

5. 攻击 E。判断：同意。这一点我在 Round 1 确实低估了。proposal 目前最大的拒稿风险，不是"实验做不出来"，而是"实验做出来了，但结果太顺、太 obvious"。如果最后是 direct prediction 漏得很多、authority 几乎不漏、CPT 只有 real-outcome、没有任何反常模式，那就是一篇执行不错但 punch 不够的论文。修补方案是反常识地"不要追求过于干净的二元分离"，而要优先寻找 mechanism-bearing 的复杂形状：authority 也会漏，但程度更低；matched/template controls 会缩小但不消灭差距；false-outcome 会在 prox task 上 sign-flip 而在 grounded task 上近零。pilot 阶段最该看的是 effect shape，不是样本量推进。

3. 给 authority 加一个简单 cue baseline，并扩大 CLS 之外的近重复检索。
4. 明确缩窄 claim scope，且最好加一个第二模型的轻量 replication。
