## High-priority issues

1. LOW-4's "per-field perturbation" does not independently prove that
   `members` participates in `pcsg_pair_registry_hash`.
   - file: `tests/r5a/test_finalize_run_manifest.py:1119`,
     `tests/r5a/test_finalize_run_manifest.py:1121`,
     `tests/r5a/test_finalize_run_manifest.py:1122`,
     `src/r5a/fleet.py:160`, `src/r5a/fleet.py:166`
   - Why: the only capacity-case perturbation changes `role`, `early`,
     `late`, and `members` in one override. If a future change dropped
     `members` from the canonical payload at `src/r5a/fleet.py:166`, this
     test would still pass because `role` / `early` / `late` changed the
     hash. That means the PR2 test claims per-field hash coverage but does
     not actually close the `members` part of LOW-4.
   - Suggested fix: add a capacity-pair baseline and mutate only
     `members` (for example replace one member or change member ordering,
     depending on whether order is intended to be canonical). Also consider
     deriving the perturbation map from `PCSGPair.model_fields` so future
     fields cannot be added without a matching hash-coverage decision.
